SECTION 1 — PROJECT OVERVIEW
──────────────────────────────────────────────────────────────────────
"Goodness Trade Lens" is a Progressive Web App (PWA) built as a
beginner-focused Forex educational analytics engine. It is NOT a
live brokerage or trading platform. No real trades are executed.
No real money is involved.

The project follows a strict monorepo layout managed by pnpm workspaces:

  artifacts/
    api-server/       ← Express 5 backend (port 8080)
    goodness-trade-lens/ ← React + Vite frontend (proxied at /)

  lib/
    db/               ← Drizzle ORM schema (PostgreSQL, unused in MVP)
    api-spec/         ← OpenAPI contract + Orval codegen

All traffic is routed through a Replit-managed reverse proxy:
  /api/* → api-server (port 8080)
  /      → goodness-trade-lens frontend


SECTION 2 — BACKEND API ROUTES (artifacts/api-server)
──────────────────────────────────────────────────────────────────────
Route 1: GET /api/market-data
  Purpose   : Returns mock price data for EUR/USD, GBP/USD, EUR/SGD.
              Each record includes: pair, price, change24h, spreadPips,
              closingPrices (14 data points for the SVG chart).
  Security  : No authentication required (public educational data).
  Risk      : Data is static mock data. In a production upgrade, this
              route should connect to a real market data provider (e.g.
              Alpha Vantage, OANDA) behind a server-side API key — never
              expose 3rd-party market API keys to the frontend.

Route 2: GET /api/economic-calendar
  Purpose   : Returns 7 calendar events (NFP, BOE, ECB, CPI, etc.) with
              title, currency, date, impact level, forecast, and description.
  Security  : Static data, no secrets involved. Safe.
  Risk      : Dates are hardcoded. For production, connect to a real
              economic calendar API (e.g. Forex Factory, Finnhub) via a
              server-side fetch — again, never in frontend code.

Route 3: POST /api/ai-analyze
  Purpose   : Accepts { currencyPair, technicalCondition } and calls an
              AI provider to return a beginner-friendly market analysis.
  Provider  : Tries Groq first (llama3-8b-8192), falls back to
              OpenRouter (meta-llama/llama-3.1-8b-instruct).
  Security  : GROQ_API_KEY and OPENROUTER_API_KEY are read exclusively
              from process.env on the server. They are NEVER sent to or
              accessible by the frontend client. ✓ SECURE.
  Risk      : 20-second timeout per provider. If both providers fail
              simultaneously (e.g. during OpenRouter outages), the
              endpoint returns HTTP 502. Consider adding a graceful
              "AI unavailable — try again later" message in the UI for
              production (already implemented in the frontend).
  AI Output : The AI is constrained by a system prompt that:
              (a) prohibits non-trading topics
              (b) mandates beginner-friendly language
              (c) requires explicit risk warnings
              (d) forbids guaranteed profit claims

SECTION 5 — SECURITY FINDINGS
──────────────────────────────────────────────────────────────────────
[PASS] API keys are server-side only — no exposure risk
[PASS] No SQL injection vectors (no user-controlled DB queries in MVP)
[PASS] No XSS vectors — AI text is rendered as React text nodes, not
       dangerouslySetInnerHTML
[PASS] CORS not overly permissive — Express default (same-origin)
[PASS] No sensitive data stored in localStorage or sessionStorage
[INFO] No authentication layer — app is fully public (by design for
       an educational demo). Add Replit Auth before any account features.
[INFO] Timeout on AI providers is 20s — acceptable for educational use,
       but consider reducing to 10s with a retry UI for production.
[INFO] No rate limiting on /api/ai-analyze — add express-rate-limit
       before public launch to prevent API key exhaustion.


SECTION 6 — PERFORMANCE NOTES
──────────────────────────────────────────────────────────────────────
- SVG chart uses preserveAspectRatio="none" — renders efficiently at
  any viewport without JavaScript resize listeners.
- Market data and calendar are fetched once on mount (no polling).
  For real-time prices, add a 10-second setInterval refetch gated on
  document.visibilityState === 'visible'.
- Service worker caches static assets (HTML, JS, CSS) — subsequent
  loads are near-instant offline.
- backdrop-filter (glassmorphism) is GPU-accelerated on modern devices
  but can degrade on low-end Android. Test on target device range.
- Tailwind v4 purges unused CSS at build time — bundle size is minimal.
